## 📊 使用方法

### 方式一：Web 界面使用

#### 步骤 1: 启动服务

```bash
# Docker 方式
docker-compose up -d

# 本地运行
python main.py
```

#### 步骤 2: 访问界面

```
浏览器访问：http://localhost:8001
```

#### 步骤 3: 上传文件

1. 点击"选择文件"或拖拽文件
2. 上传包含商品数据的 Excel 文件
3. 系统自动识别表头和数据

#### 步骤 4: 选择模板

在"导出配置"区域：
1. **ERP模板** 下拉框选择：`思迅天店（模板文件）`
2. 填写 **供应商编码**（必填）
3. 填写 **商品类别**（必填）

#### 步骤 5: 导出文件

点击 **"📥 导出 Excel"** 按钮，下载生成的导入文件。

#### 步骤 6: 导入系统

1. 登录思迅天店系统
2. 进入商品管理 → 商品导入
3. 上传导出的 Excel 文件
4. 完成导入

---

### 方式二：API 调用使用

#### Python 示例

```python
import requests

# 1. 上传源文件
upload_url = "http://localhost:8000/api/upload"
files = {"file": open("商品数据.xlsx", "rb")}
response = requests.post(upload_url, files=files)
result = response.json()

# 2. 导出（使用天店模板文件）
export_url = "http://localhost:8000/api/export"
export_data = {
    "results": result['results'],
    "manual_codes": {
        "supplier_code": "SUP001",      # 供应商编码
        "category_code": "CAT001"       # 商品类别
    },
    "template_file": "moban/天店商品导入模板.xlsx"
}
export_response = requests.post(export_url, json=export_data)

# 3. 保存导出文件
with open("天店导入文件.xlsx", "wb") as f:
    f.write(export_response.content)

print("✅ 导出成功！文件：天店导入文件.xlsx")
```

#### cURL 示例

```bash
# 1. 上传文件
curl -X POST "http://localhost:8000/api/upload" \
  -F "file=@商品数据.xlsx" \
  -o upload_result.json

# 2. 导出（需要从 upload_result.json 中提取 results）
curl -X POST "http://localhost:8000/api/export" \
  -H "Content-Type: application/json" \
  -d '{
    "results": [...],
    "manual_codes": {
      "supplier_code": "SUP001",
      "category_code": "CAT001"
    },
    "template_file": "moban/天店商品导入模板.xlsx"
  }' \
  --output 天店导入文件.xlsx
```

## 🔧 字段映射说明

### 自动映射字段

系统会自动识别并映射以下字段：

| 源文件字段 | 模板字段 | 关键词 |
|-----------|---------|--------|
| 条码 | 商品编码 | 条码、条形码、商品码、货号、编码、code、barcode |
| 品名 | 商品名称 | 品名、商品名、商品全名、产品名、名称、商品 |
| 规格 | 规格型号 | 规格、型号、容量、净含量 |
| 单位 | 计量单位 | 单位、包装、计量单位 |
| 进价 | 进货价 | 进价、成本价、批发价、采购价、单价、cost |
| 零售价 | 销售价 | 零售价、售价、零售金额、销售价、价格、retail |

### 手工填写字段

导出时需要在界面填写：

| 字段 | 说明 | 是否必填 |
|------|------|---------|
| 供应商编码 | 供应商代码，如：SUP001 | ✅ 必填 |
| 商品类别 | 商品分类代码，如：CAT001 | ✅ 必填 |

### 可选字段

以下字段如果源文件中存在，会自动映射；不存在则留空：

- 品牌
- 产地
- 保质期(天)
- 备注

## ⚙️ 高级配置

### 自定义字段映射

如果自动映射不准确，可以手动指正：

1. 在识别结果页面找到 **"手动指正配置"** 区域
2. 为每个字段选择正确的列
3. 点击 **"应用指正"** 重新提取
4. 确认数据无误后导出

### 修改模板文件

如需调整模板字段：

1. 打开 `moban/天店商品导入模板.xlsx`
2. 修改第一行的字段名
3. 保存文件
4. 重新导出即可

**注意**：修改后需确保字段名与思迅天店系统要求一致。

## ❓ 常见问题

### Q1: 导出后字段缺失怎么办？

**A**: 可能原因及解决方法：
- **原因**：源文件中没有对应的列
- **解决**：使用"手动指正"功能选择正确的列

### Q2: 供应商编码和商品类别在哪里填写？

**A**: 
- 在导出配置区域的输入框中填写
- 或在 API 调用时通过 `manual_codes` 参数传递

### Q3: 模板文件可以修改吗？

**A**: 
- ✅ 可以修改第一行的字段名
- ✅ 可以调整列宽、样式
- ⚠️ 不要删除必填字段
- ⚠️ 字段名必须与思迅天店系统一致

### Q4: 如何验证导出文件是否正确？

**A**: 
1. 打开导出的 Excel 文件
2. 检查字段名是否与模板一致
3. 检查数据是否完整
4. 在思迅天店系统中测试导入

### Q5: 支持批量导入吗？

**A**: 
- ✅ 支持，建议单次不超过 10,000 行
- 大数据量可分批处理

## 🔍 故障排查

### 问题 1: 找不到模板文件

**错误信息**：
```
模板文件不存在: moban/天店商品导入模板.xlsx
```

**解决方法**：
```bash
# 检查文件是否存在
ls -la /home/data/smart_excel/moban/天店商品导入模板.xlsx

# 如果不存在，请重新上传文件
```

### 问题 2: 字段映射错误

**现象**：导出的数据列错位

**解决方法**：
1. 使用"手动指正"功能
2. 检查源文件表头是否标准
3. 确认列名关键词是否正确

### 问题 3: 导入系统失败

**现象**：思迅天店系统提示格式错误

**解决方法**：
1. 检查必填字段是否完整
2. 确认供应商编码和类别编码是否正确
3. 验证数据格式（如：条码长度、价格格式）

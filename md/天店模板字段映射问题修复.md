# 天店模板文件字段映射问题修复记录

## 问题描述

使用"思迅天店（模板文件）"导出时，数据无法正确映射到模板字段，导出的 Excel 列名错误、数据错位。

---

## 排查过程

### 第一步：读取天店模板文件结构

通过 Docker 容器中的 Python 读取模板文件：

```python
df = pd.read_excel('/app/moban/天店商品导入模板.xlsx')
```

**发现**：模板文件不是标准的第一行即字段名格式。

| 行号 | 内容 | 类型 |
|------|------|------|
| 第1行 | 大段说明文本（合并单元格） | 说明行 |
| 第2行 | 货号、品名、自编码、规格... | **实际字段名** |
| 第3行+ | 6901028137492, 七匹狼(红)条... | 数据行 |

天店模板的 **24 个实际字段**：
```
货号、品名、自编码、规格、单位、类别编码、类别、品牌、
供应商编码、供应商、进货价、零售价、会员价、最低售价、
批发价、商品状态、计价方式、产地、进货规格、联营扣率、
配送价、商品类型、允许积分、允许折扣
```

### 第二步：分析导出器代码逻辑

定位到 `core/exporter/excel_exporter.py` 中的 `export_by_template_file` 方法：

```python
# 原代码 - 问题所在
template_df = pd.read_excel(template_file)          # header=0，读到了说明行
field_order = template_df.columns.tolist()           # 得到 Unnamed:1, Unnamed:2...
```

以及 `_merge_data_by_template` 方法：

```python
# 原代码 - 问题所在
FIELD_MAPPING = {
    "条码": "条码",      # 但天店模板字段名是 "货号"
    "品名": "名称",      # 但天店模板字段名是 "品名"
    "进价": "进价",      # 但天店模板字段名是 "进货价"
    ...
}
```

### 第三步：确认两个根因

| 问题 | 原因 | 影响 |
|------|------|------|
| **表头行错误** | `pd.read_excel(template_file)` 默认 `header=0`，读到说明行 | 列名全是 `Unnamed:1, Unnamed:2...` |
| **字段映射硬编码** | `FIELD_MAPPING` 写死目标字段名，与模板实际字段不匹配 | 数据写到错误的列或丢失 |

---

## 修复方案

### 修复1：自动检测表头行

新增 `_detect_template_header_row()` 方法，扫描前 10 行，通过关键词匹配找到真正的表头行：

```python
def _detect_template_header_row(self, template_file: str) -> int:
    header_keywords = ['货号', '品名', '条码', '名称', '规格', '单位', ...]
    for header_row in range(10):
        df = pd.read_excel(template_file, header=header_row, nrows=1)
        # 过滤无效列名，检查关键词匹配数
        if match_count >= 2 and len(valid_cols) >= 3:
            return header_row
    return 0
```

### 修复2：基于关键词的智能字段匹配

新增 `SEMANTIC_KEYWORDS` 语义映射表和 `_match_field_to_template()` 方法：

```python
SEMANTIC_KEYWORDS = {
    "条码":   ["货号", "条码", "条形码", "商品码", "编码", ...],
    "品名":   ["品名", "名称", "商品名", "商品名称", ...],
    "进价":   ["进价", "进货价", "成本价", "采购价", ...],
    "零售价": ["零售价", "售价", "销售价", "销售单价", ...],
    ...
}
```

匹配逻辑：对模板中的每个字段名，检查是否包含某语义的关键词，实现自动对应。

### 修复3：重写数据合并逻辑

`_merge_data_by_template()` 不再使用硬编码映射，改为：

1. 预计算每个语义字段匹配到哪个模板字段
2. 按匹配结果写入数据
3. 手工编码字段也通过关键词匹配到模板字段名
